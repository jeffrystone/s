### Как читать статусы под требованиями ТЗ

Подсветка: ниже среди маркированных списков **пункты без ведущего** `~~…~~` обёрнуты в `<span style="color:#991b1b;">…</span>`. Встроенный предпросмотр Markdown (VS Code / Cursor) **снимает с `<span>` обычные `class`** (разрешён только паттерн codicon) и оставляет **только узко разрешённый** `style` — форма вроде `**color:#991b1b;`** (**обязательна** финальная `;` во значении атрибута). Иначе оформление не попадает в DOM и строка выглядит цветом темы («всё белым» на тёмной теме).

- Если требование **выполнено целиком** — текст зачёркнут целиком `~~…~~`, либо все подпункты под ним закрыты.
- Если выполнено **частично** — описание задачи не зачёркнуто или зачёркнуто не полностью; под ним **подсписок подзадач**:
  - `~~…~~` — подзадача уже есть в коде;
  - обычный маркер — подзадача ещё нет или сильнее упрощена, чем в ТЗ;
  - строка с литералом `**Блокер:`** — внешняя или архитектурная причина, из‑за которой оставшиеся подзадачи нельзя честно завершить без новой работы (API, клиент, отсутствующий сбор метрик).
